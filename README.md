# Lecture Transcript Refiner

Refine existing lecture, interview, and subtitle transcripts: correct dictation errors, homophones, ASR text mistakes, punctuation, and paragraphing, then optionally transform the corrected transcript into structured written lecture notes for Obsidian.

This skill does not perform ASR by itself. It works with transcripts generated by tools such as Whisper, meeting recorders, subtitle exporters, or other speech-to-text services.

## What It Does

- Corrects dictation errors, homophones, typos, punctuation, and paragraphing in lecture-style transcripts.
- Preserves the speaker's wording, tone, sequence, examples, and argument chain during proofreading.
- Supports a strict mode for users who want to keep the spoken style as close to the original as possible.
- Supports a two-pass note workflow: proofread first, then rewrite into a structured Obsidian lecture note.
- Adds metadata, source links, and reusable lecture-review sections when generating Obsidian notes.

## Best For

- Lecture transcripts
- Interview transcripts
- Meeting or seminar transcripts
- Subtitle text
- Voice-to-text drafts that are already transcribed but still messy

## Not For

- Direct audio or video transcription
- General article polishing
- Essay grammar checking
- Generic rewriting unrelated to transcript cleanup
- Inbox routing, webpage capture, or media downloading

## Files

- `SKILL.md`: The skill's routing and execution instructions.
- `prompt.md`: The proofreading and strict-mode prompt.
- `prompt_rewrite.md`: The written lecture-note prompt.
- `test-prompts.json`: Example prompts for evaluating the skill.

## Suggested Workflow

1. Generate a transcript with your preferred ASR or meeting-recording tool.
2. Use this skill to correct dictation and homophone errors.
3. If needed, ask it to turn the corrected transcript into a written Obsidian lecture note.
4. Review uncertain names, dates, and speaker metadata before sharing or publishing the note.
