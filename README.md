# Lecture Transcript Refiner

把已有的讲座、访谈、字幕转写稿整理成更准确、更可读、也更适合沉淀的文字材料。

它可以先做听写校正：修正同音字、错别字、ASR 文本错漏、标点和分段。
如果你用了 Obsidian，它还可以进一步把校正后的讲座稿整理成结构化的书面讲座笔记。

这个项目特别适合这样的用户：

- 手里已经有录音转文字、会议转写稿或字幕文本
- 想把粗糙转写稿整理成可读文稿
- 想修正“立德树人 / 利得数人”“构建 / 勾结”这类同音误识别
- 想把讲座内容沉淀成 Obsidian 笔记、备课资料、教研资料或写作素材

## 能做什么

这个技能主要做两件事。

第一，听写校正：

- 修正语音识别造成的同音字、近音词和错别字
- 修正教育、AI、教学研究等场景里的常见术语误识别
- 补全标点和分段，让讲稿更好读
- 删除少量无意义口头禅，但保留讲者原本的措辞、语气和例子
- 支持“尽量原话”的严格校对模式

第二，书面整理：

- 在校正稿基础上整理成一篇结构化讲座回顾
- 提炼讲座主旨、关键案例、方法框架和可复用素材
- 特别关注 AI 教学案例、课堂机制、工具使用、评价方式和人机协作启发
- 输出适合 Obsidian 的 Markdown 笔记
- 自动加入 YAML 元信息和原始校正稿链接

## 适合谁

这个技能优先面向：

- 高校教师
- 教育技术研究者
- 研究生
- 教研人员
- 使用 Obsidian 做知识管理的人
- 经常整理讲座、访谈、会议记录的人

## 开始前先知道

这是一个“转写稿后处理”技能。

它适合：

- 已经转成文字的讲座稿
- 已经转成文字的访谈稿
- 字幕文本
- 会议或研讨活动的录音转文字结果
- 需要校正同音字和 ASR 错漏的文本草稿

它不适合：

- 直接把音频转成文字
- 直接把视频转成字幕
- 普通文章润色
- 论文语法检查
- 和讲座/访谈/录音转文字无关的泛泛改写
- 网页抓取、媒体下载、入库分流

如果你还没有文字稿，需要先用 Whisper、飞书妙记、腾讯会议、剪映、讯飞、通义听悟、YouTube 字幕或其他 ASR 工具生成转写稿，再使用这个技能。

## 安装方式

### 第 1 步：下载项目

如果你会用 Git：

```bash
git clone https://github.com/ranrancc/lecture-transcript-refiner.git
cd lecture-transcript-refiner
```

如果你不会用 Git：

1. 打开这个仓库首页
2. 点击 `Code`
3. 点击 `Download ZIP`
4. 解压后进入项目文件夹

### 第 2 步：放到你的技能目录

如果你使用 Codex / OpenClaw 这一类支持本地 skills 的工具，可以把这个文件夹放到你的 skills 目录里。

例如：

```bash
mkdir -p ~/.codex/skills
cp -R lecture-transcript-refiner ~/.codex/skills/
```

如果你的工具使用的是其他技能目录，请放到对应位置，并确保 `SKILL.md`、`prompt.md`、`prompt_rewrite.md` 在同一个文件夹里。

## 常见使用方式

### 做听写校正

```text
请把 /path/to/20260304 苏涛 AI时代的大学课堂.txt 做听写校正。
```

这会优先做：

- 同音字纠正
- 专有名词修复
- 标点和分段整理
- 轻微口头噪声清理

输出文件通常会保存在原文件旁边，并在文件名里加上 `（已校正）`。

### 尽量保留原话

```text
这份讲座转写稿已经校过一遍了，请保留口语风格，尽量原话，只修错别字和标点。
```

这种情况下会进入更保守的严格校对模式：

- 不主动书面化
- 不合并大段内容
- 不提炼成摘要
- 尽量保留讲者原来的句式和语气

### 整理成 Obsidian 讲座笔记

```text
把这个录音转文字书面整理成 Obsidian 讲座回顾笔记。
```

这种情况下会走两步：

1. 先把原始转写稿做听写校正
2. 再把校正稿整理成结构化讲座回顾笔记

讲座笔记会尽量包含：

- 元信息与主旨判断
- 最值得保留的 AI 教学案例与做法
- 高价值素材库
- 对教育技术研究和 AI 教学实践的启发
- 后续写作或研究转化线索

## 推荐工作流

1. 先用你熟悉的工具生成转写稿
2. 用这个技能做听写校正
3. 人工快速检查讲者姓名、机构、日期、术语和关键数字
4. 如果这场讲座值得沉淀，再让它整理成 Obsidian 讲座笔记
5. 最后根据你的真实理解做二次修订

## 新手排错

### 1. 它是不是可以直接识别音频？

不可以。它不包含 ASR。

你需要先把音频或视频转成文字，再把文字稿交给这个技能处理。

### 2. 为什么它没有生成 Obsidian 笔记？

常见原因是你的指令只说了“校正”“整理语音转文字”，没有明确说“书面整理”“讲座回顾”或“Obsidian 笔记”。

如果你想要笔记，可以直接说：

```text
请先校正这份转写稿，再整理成 Obsidian 讲座回顾笔记。
```

### 3. 为什么有些人名、机构名还是不确定？

转写稿里的人名、机构名和课程名可能本身就缺少上下文。这个技能会尽量避免编造信息。

建议你在原文或指令里补充：

- 讲者姓名
- 讲者身份
- 讲座标题
- 讲座日期
- 活动场合

### 4. 很长的转写稿怎么处理？

技能说明里已经包含分块处理规则。

如果文本特别长，建议保留原始文件路径，让工具按文件处理，不要一次性粘贴到对话框里。
